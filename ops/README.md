# Operations System

This folder defines the repeatable local operating system for CVAGENT upgrades.

- `testing/`: test plans, verification checklists, and run summaries.
- `updates/`: change records for each product or workflow upgrade.
- `debug/`: issue-driven debugging records using the sequence: problem record -> code review -> fix -> resolved status.

Private inputs, resumes, JD images, API responses, and generated resume outputs belong under `.local_user_archive/` or `.local_runs/` and must not be committed.
